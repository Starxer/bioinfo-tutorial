## 1. 测序技术三十年：从 Sanger 到三代

![[imgs/metaso-sanger-ngs-3.jpg]]

### Sanger 测序（第一代，1977）

- **原理**：双脱氧核苷酸（ddNTP）链终止法
- DNA 聚合酶合成互补链时，随机掺入带荧光的 ddNTP → 链终止
- 产生不同长度的片段 → 毛细管电泳分离 → 荧光检测 → 读出序列
- **一次跑一条序列**，长度约 800–1000 bp
- **优点**：准确率极高（>99.9%），至今仍是"金标准"
- **缺点**：通量太低，不适合大规模测序

> 用来做：验证突变、小片段测序。人类基因组计划（2003）主要靠这个，花了 13 年 30 亿美元。

### 二代测序 NGS（高通量测序，2005–2007）

里程碑：454（2005）→ Solexa/Illumina（2007）→ SOLiD

**Illumina 是目前最主流的平台**，原理用三个步骤概括：

1. **文库制备**：把 DNA 打碎成 200–600 bp 的片段，两端加上接头（Adapter）
2. **簇生成**：在 flow cell 上进行桥式 PCR（Bridge Amplification），每个 DNA 片段扩增成一个个簇（Cluster），约 1000 个拷贝
3. **边合成边测序（SBS）**：每次加入一个带荧光标记的核苷酸，合成延伸后拍照，四色荧光对应 A/T/C/G

![[imgs/illumina-flowcell.jpg]]

> 上图：Illumina 流动池（Flow Cell）实物照片，8 条泳道（Lane）清晰可见。每条 Lane 内可以同时测数百万个簇。

- **一次跑几亿条序列**（150 bp × 2 即 PE150，是最常见的模式）
- 人类基因组现在只需**一天、几百美元**
- 准确率 ~99.9%（Q30 水平）

> 主流生信流程处理的数据 90% 以上是 Illumina 产出。

### 三代测序（2010s 至今）

PacBio HiFi（长读长，~15–25 kb）和 Oxford Nanopore（超长读长，可达 Mb 级别）

- **不是"更好"，而是不同**：长读长可以直接跨越重复区域，解决基因组组装难题
- 准确率近年已经追上 NGS（HiFi > Q30）
- 价格仍然较高，适合组装基因组、鉴定结构变异

## 2. FASTQ 和 FASTA：生信最基础的两种序列格式

生信世界里你打交道最多的就是这两种格式——FASTQ 存原始测序数据，FASTA 存参考序列。

### FASTQ：测序下机数据

所有测序下机数据，第一步拿到的就是 `.fastq` 文件（或压缩的 `.fastq.gz`）。

![[imgs/fastq-format.svg]]

每条序列固定 **4 行**为一组：

```
@A00265:1178:H357TDSX7:1:1101:1904:1000 1:N:0:GGACTCCT+TATGCAGT
GAAGAAACATCTGCTGTGGTGCGCAGCGTTTGCTCGGCAGCGATAGCTTTCTTGTATTGCTGCAACTTTTCAGAC
+                                                         ← 分隔符
FFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFFF:FF:F,FFFF,,FFF
```

> 以上是真实数据：家蚕（*Bombyx mori*）Illumina PE150 测序的第一个 read。第4行每个字符代表对应位置碱基的质量——`F` 表示高质量（Q37），`,` 和 `:` 稍低但仍然可接受。

**四行分别是什么：**

| 行 | 标识 | 含义 | 示例 |
|----|------|------|------|
| 1 | `@` | 序列 ID + 测序仪信息 | `@A00265:...` |
| 2 | 碱基 | A/T/C/G/N 组成的序列 | `GAAGAAAC...` |
| 3 | `+` | 分隔符（有时重复 ID） | `+` |
| 4 | 质量 | 每个碱基一个 ASCII 字符 | `FFFFF:FF...` |

**质量分数（Phred Quality Score）**：第4行每个字符对应一个 Q 值

```
Q = -10 × log₁₀(P)    P = 碱基出错的概率
```

| Q 值 | 碱基正确率 | 出错概率 | 对应 ASCII | 质量判断 |
|------|----------|---------|-----------|---------|
| Q40  | 99.99% | 1/10000 | `I` | 极好 |
| Q30  | 99.9% | 1/1000 | `?` | 良好 |
| Q20  | 99% | 1/100 | `5` | 及格 |
| Q10  | 90% | 1/10 | `+` | 差 |

> **实用经验**：Q30 ≥ 80% 的测序数据质量不错。用 **FastQC** 可以快速查看质量报告。测序数据通常以 `.fastq.gz` 压缩存储——不需要解压，生信工具都能直接读 gzip。

### FASTA：只存序列，没有质量

FASTA 是比 FASTQ 更简单、更古老的格式——**只有序列，没有质量信息**。参考基因组、基因序列、蛋白质序列都用它。

```
>NC_051358.1 Bombyx mori strain p50T chromosome 1, Bmori_2016v1.0
ctaacctaacctaacctaacctaacctaacctaacctaacctaacctaacctaacctaac
ctaacctaacctaacctaacctaacctaacctaacctaacctaacctaacctaacctaac
>NC_051359.1 Bombyx mori strain p50T chromosome 2, Bmori_2016v1.0
CCCGTATTTACAGTTTCTATCAATTTTTTCTGTCTTTGCACACCCATTTTTCCAATTTCA
ATCGCCCGTTTCTTTTCAAAACTTCTTCATTCTTCTTCGCCAACGTTTTCCACACCCTTT
```

> 以上是家蚕参考基因组（GCF_014905235.1）的前两条染色体。每行 60 个碱基是常见约定。

**FASTA 的规则：**

- `>` 开头的一行是**序列描述**（header），紧跟序列本身
- 一个文件可以有多条序列（multi-FASTA）——每条以 `>` 开始
- 扩展名多样：`.fa` `.fasta` `.fna`（核酸） `.faa`（氨基酸）

**FASTA vs FASTQ：**

| | FASTQ | FASTA |
|---|-------|-------|
| 用途 | 测序原始数据 | 参考序列、基因序列 |
| 质量 | **有**（第4行） | **无** |
| 行/序列 | 固定 4 行 | 不固定（序列可任意换行） |
| 你用它… | 跑比对之前 | 作为比对的"靶子" |

## 3. 常见生信文件格式

从原始数据到分析结果，你会反复遇到这些文件：

| 格式 | 扩展名 | 用途 | 一句话记住 |
|------|--------|------|-----------|
| **FASTQ** | `.fastq` | 序列 + 质量 | 测序仪直接输出的格式 |
| **SAM** | `.sam` | 比对结果（人类可读） | "序列打到参考基因组上的结果" |
| **BAM** | `.bam` | SAM 的二进制压缩版 | 生信最常用的中间格式 |
| **GFF/GTF** | `.gff`, `.gtf` | 基因注释（外显子、CDS等） | "基因组的说明书" |
| **VCF** | `.vcf` | 变异位点（SNP/Indel） | "和参考基因组不一样的地方" |

> **一个典型 RNA-seq 流程的文件变化**：
> `.fastq`（原始数据）→ `.bam`（比对）→ `.counts`（定量）→ `.csv`（差异表达结果）

## 4. 从 Illumina 下机到 FASTQ

Illumina 测序仪生成的原始数据是图像文件（`.tif`），经过 Base Calling 软件（如 Illumina 的 bcl2fastq 或 bcl-convert）转换成 `.fastq`。

**最重要的参数：Read 长度**

```
PE150  → 每条序列读 150 bp，双向测序
PE100  → 100 bp
SE50   → 单端 50 bp
```

> PE150 是目前 RNA-seq 和 WGS 的最常见配置。PE = Paired-End（两端都读），SE = Single-End（只读一端）。

## 小结

- 三代测序：**Sanger**（一条序列，准）→ **NGS**（几亿条并行，高通量）→ **TGS**（超长读长）
- Illumina NGS 三步：文库制备 → 簇生成 → 边合成边测序
- FASTQ 每 4 行 = 一条 read（ID → 序列 → + → 质量），FASTA 用 `>` 开头
- Q30 ≥ 80% 是合格数据的参考线
- 常见文件格式：FASTQ（测序数据）→ FASTA（参考序列）→ BAM（比对）→ VCF（变异）

### 练练手

1. 找一个 `.fasta` 文件，用 `head` 看前 10 行，用 `grep ">"` 数数里面有多少条序列
2. 验证公式：如果 Q = 30，碱基出错的概率是多少？（提示：Q = -10 × log₁₀P）
3. 搜索并了解：你研究的物种，参考基因组在哪下载？用什么格式？
