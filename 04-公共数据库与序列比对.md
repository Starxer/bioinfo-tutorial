# 04 — 公共数据库与序列比对

> **学完本章你能**：从 NCBI 下载序列和基因组、用 BLAST 在线搜索相似序列、理解比对的两种基本类型（精确匹配 vs 近似匹配）

---

## 1. 三大公共数据库

| 数据库 | 网址 | 定位 | 最适合 |
|--------|------|------|--------|
| **NCBI** | https://www.ncbi.nlm.nih.gov/ | 最通用、最全面 | 找序列、文献、物种、基因 |
| **Ensembl** | https://ensembl.org/ | 脊椎动物基因组 | 已注释的基因组浏览与下载 |
| **UCSC Genome Browser** | https://genome.ucsc.edu/ | 基因组可视化浏览 | 看基因组区域、多物种比较 |

### NCBI 快速上手

NCBI 的入口是它的搜索框，类似 Google 但你搜的是生物学概念。

```
搜索框输入一个基因名 → 你会看到：
├── Nucleotide（核苷酸序列）
├── Protein（蛋白序列）
├── Gene（基因信息、位置、功能）
├── Pubmed（文献）
├── SRA（原始测序数据）
└── GEO（表达芯片/RNA-seq 数据）
```

**常用操作：**

```bash
# 从 NCBI 命令行下载序列（安装 entrez-direct）
mamba install -c bioconda entrez-direct

# 搜索基因
esearch -db gene -query "Bombyx mori BmSgAD"

# 下载序列
efetch -db nucleotide -id NM_001043364 -format fasta
```

### Ensembl 和 UCSC 什么时候用？

- **Ensembl**：想知道一个基因在基因组上具体哪个位置、有哪些转录本、外显子结构 → 打开 Ensembl 搜索基因名，看"Gene-based displays"
- **UCSC Browser**：想看某个基因组区域的可视化展示（BAM 比对、SNP、重复区域） → 打开 UCSC Browser，输入坐标 `chr1:10000-20000`

## 2. 从 NCBI 下载基因组

```bash
# 下载参考基因组（以家蚕为例）
# 方法1：wget 直接下载
wget https://ftp.ncbi.nlm.nih.gov/genomes/all/GCF/000/151/625/GCF_000151625.1_ASM15162v1/GCF_000151625.1_ASM15162v1_genomic.fna.gz

# 解压
gunzip GCF_000151625.1_ASM15162v1_genomic.fna.gz

# 配套的注释文件
wget https://ftp.ncbi.nlm.nih.gov/genomes/all/GCF/000/151/625/GCF_000151625.1_ASM15162v1/GCF_000151625.1_ASM15162v1_genomic.gff.gz
```

> FASTA 文件里 `>` 开头的是序列标题行，后面跟着序列。一个 `.fna` 文件包含整个基因组的染色体和 scaffold。

## 3. BLAST——找到和你的序列相似的序列

BLAST（Basic Local Alignment Search Tool）是最常用的序列比对工具。

### 在线 BLAST

1. 打开 https://blast.ncbi.nlm.nih.gov/
2. 把你的序列（FASTA 格式）粘贴进去
3. 选择数据库（nr = 非冗余蛋白库，nt = 核酸库）
4. 点击 BLAST

**结果怎么看：**

- **Score / E-value**：越高/越小越好。E-value < 1e-5 才算显著
- **Identity**：匹配百分比。> 90% 通常表示同源
- **Query Cover**：查询序列覆盖了目标序列多大比例

> **BLAST 的核心用途**：
> - 这条新测的序列是什么基因？
> - 这个基因在其他物种里有同源序列吗？
> - 我的序列有没有被污染（比如混进了细菌序列）？

### 本地 BLAST

数据量大时，在本地建索引比对更快：

```bash
# 安装
mamba install -c bioconda blast

# 建索引
makeblastdb -in reference.fasta -dbtype nucl

# 比对
blastn -query myseq.fasta -db reference.fasta -out results.txt
```

## 4. 序列比对：精确匹配 vs 近似匹配

这是理解生信工具背后原理的关键。

### 精确匹配（Exact Match）

找一模一样的序列。速度快，但读长、变异、测序错误会导致找不到。

```bash
# 用得少，但理解原理有用
# 想象 grep 在文本里找精确字符串
grep "ATCGATCG" genome.fasta
```

### 近似匹配（Approximate / Heuristic Match）

允许错配和 gaps（插入/缺失）。**绝大多数生信工具用的都是这个。**

**代表工具：**

| 工具 | 用途 | 比对策略 |
|------|------|---------|
| **BLAST** | 序列搜索（核酸/蛋白） | 先找短种子（seed），再扩展 |
| **BWA** | NGS 短读长比对到参考基因组 | BWT/FM-index（快速压缩索引） |
| **minimap2** | 长读长/跨物种比对 | 基于 minimizer 的种子索引 |
| **STAR** | RNA-seq 比对（考虑剪接） | 先找最大可匹配种子（Maximal Mappable Prefix） |

**一个简单的类比**：

```
精确匹配：在字典里找"apple"——一模一样才能找到
近似匹配：在字典里找"aple"（拼错了）→ 也给你指出 apple、maple
```

> **BWA 和 STAR 是生信最常用的两个比对工具**。
> BWA 做 DNA 比对（WGS、WES），STAR 做 RNA-seq 比对（因为它能处理内含子剪接）。

## 小结

- 三大数据库：NCBI（最通用）、Ensembl（脊椎动物基因组）、UCSC Browser（可视化和浏览）
- BLAST 是在线搜索相似序列的核心工具，看 E-value 和 Identity
- 精确匹配 = 一模一样；近似匹配 = 允许错配和 gaps——大多数生信工具用近似匹配
- BWA（DNA 短读长）和 STAR（RNA-seq 考虑剪接）是最常用的比对工具

### 练练手

1. 打开 [NCBI BLAST](https://blast.ncbi.nlm.nih.gov/)，粘贴一段你熟悉的基因序列，看看结果里最相似的是什么物种
2. 用 `esearch` 查找你研究物种的一个基因，用 `efetch` 下载它的 CDS 序列
3. 思考：为什么 RNA-seq 比对需要 STAR 而不是 BWA？
